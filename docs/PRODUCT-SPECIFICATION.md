## 一、系统概述

### 1.1 产品定位

VANZO KOL 营销 Agent 是一款**AI 原生的营销决策辅助系统**，旨在帮助品牌方和营销团队实现 KOL 投放的全链路自动化。系统通过四个专业化 Agent 的协作，从需求分析到效果优化，实现**分钟级**的营销决策输出。

### 1.2 核心价值主张

| 价值点 | 传统方式 | 本系统 |
|--------|----------|--------|
| 决策周期 | 3-5 个工作日 | 15-30 分钟 |

| 效果预测 | 经验估算 | 数据驱动预测 |

## 二、Layer 1：需求分析 Agent（Requirement Analyzer）

### 2.1 Agent 定位

需求分析 Agent 是整个系统的**入口 Agent**，负责将用户的自然语言需求或结构化输入转化为标准化的营销需求文档（Marketing Requirement Document, MRD）。它是整个链路的"大脑"，决定了后续所有决策的准确性。

### 2.2 核心功能模块

#### 2.2.1 品牌调性分析（Brand Tone Analysis）

**功能描述**：从品牌名称、描述、产品特性等信息中提取品牌的核心调性关键词。

**输入字段**：
- 品牌名称（必填）
- 品牌描述（必填，50-500 字）
- 现有品牌素材（可选，图片/视频 URL）
- 品牌历史投放数据（可选）

**处理流程**：
```
1. 文本预处理 → 分词 + 去除停用词
2. 关键词提取 → TF-IDF + 品牌领域词库
3. 情感分析 → 正面/中性/负面情感倾向
4. 调性归类 → 高端/大众/年轻/专业/亲和 等维度打分
5. 输出结构化调性报告
```

**输出字段**：
```json
{
  "brandTone": {
    "primary": "高端",
    "secondary": ["专业", "创新"],
    "scores": {
      "高端": 0.85,
      "年轻": 0.45,
      "专业": 0.78,
      "亲和": 0.32,
      "大众": 0.28
    },
    "keywords": ["品质", "创新", "科技", "轻奢"],
    "avoidKeywords": ["便宜", "低端", "折扣"],
    "toneDescription": "面向都市精英的高端科技品牌"
  }
}
```

#### 2.2.2 受众画像生成（Audience Profiling）

**功能描述**：基于品牌和产品特性，生成目标受众的详细画像。

**输入**：
- 产品品类
- 目标市场（国家/地区）
- 价格区间
- 竞品信息（可选）

**处理流程**：
```
1. 品类知识检索 → 查找品类典型受众特征
2. 市场画像匹配 → 结合目标市场人口统计
3. 价格敏感度分析 → 基于价格区间推断消费能力
4. 竞品受众分析 → 识别差异化定位机会
5. 生成多维度受众画像
```

**输出字段**：
```json
{
  "targetAudience": {
    "primary": {
      "ageRange": "25-35",
      "gender": "女性为主",
      "income": "月入 15K-30K RMB",
      "location": "一二线城市",
      "interests": ["美妆", "护肤", "生活方式"],
      "painPoints": ["肌肤敏感", "追求效率"],
      "purchaseBehavior": "线上种草，直播间转化"
    },
    "secondary": {
      "ageRange": "20-28",
      "characteristics": "学生党和初入职场"
    }
  }
}
```

#### 2.2.3 竞争环境评估（Competitive Landscape）

**功能描述**：分析目标市场的竞争格局，识别机会点和风险点。

**处理能力**：
- 品类热度分析（搜索量、社交讨论量）
- 竞品投放策略分析
- KOL 竞争度评估（头部 KOL 档期、报价）
- 市场价格区间调研

**输出字段**：
```json
{
  "competitiveAnalysis": {
    "categoryHeat": "高",
    "marketTrend": "上升",
    "competitors": [
      {"name": "竞品A", "投放预算": "50W/月", "主推KOL类型": "头部+腰部"}
    ],
    "opportunities": ["竞品忽视的垂类KOL", "新兴内容形式"],
    "risks": ["头部KOL报价上涨", "同质化内容扎堆"]
  }
}
```

#### 2.2.4 合规检查（Compliance Check）

**功能描述**：确保营销需求符合目标市场的法律法规和平台规则。

**检查项目**：
- 广告法合规（禁用词、极限词）
- 平台社区规范（TikTok/Instagram/小红书 等）
- 跨境合规（海关、税务、知识产权）
- 行业特殊规定（美妆、食品、医药）

## 三、Layer 2：KOL 匹配 Agent（KOL Matcher）

### 3.1 Agent 定位

KOL 匹配 Agent 是系统的**核心价值输出 Agent**，负责从海量 KOL 数据库中筛选出与需求最匹配的候选人。它决定了投放的效率和效果上限。

### 3.2 核心功能模块

#### 3.2.1 KOL 数据库检索

**数据规模**：
- 10M+ KOL 基础信息（全球主流平台）
- 100M+ 互动数据（最近 12 个月）
- 50M+ 内容数据（主题分类、标签）

**数据字段**：
```json
{
  "kol": {

    "name": "美妆博主 Lily",
    "platforms": ["tiktok", "instagram", "xiaohongshu"],

    },

      "tiktok": 0.065,
```

#### 3.2.2 多维匹配评分算法

**评分维度及权重**：

| 维度 | 权重 | 说明 |
|------|------|------|
| 受众匹配度 | 30% | KOL 粉丝画像与目标受众的契合度 |
| 内容相关性 | 25% | KOL 内容主题与品牌/产品的匹配度 |
| 性价比 | 20% | 基于历史数据的 ROI 预估 |
| 互动质量 | 15% | 评论区活跃度、粉丝粘性 |
| 合作风险 | 10% | 舆情风险、档期可靠性 |

**匹配分数计算**：
```
MatchScore = Σ (维度分数 × 权重) × 风险系数

其中：
- 受众匹配度 = cosine_similarity(KOL受众向量, 目标受众向量)
- 内容相关性 = KOL内容标签与品牌标签的 Jaccard 相似度
- 性价比 = 预测CPM / 历史CPM基准 × 互动率提升系数
- 互动质量 = (评论率 + 转发率 + 收藏率) / 3
- 风险系数 = 1.0 - 风险扣分 (最低 0.5)
```

#### 3.2.3 组合优化算法

**目标**：在预算约束下，选择最优的 KOL 组合。

**优化目标**：
```
最大化: Expected_Reach × Engagement_Rate - Cost
约束条件: Total_Cost ≤ Budget
         KOL_Count ∈ [Min_Count, Max_Count]
         Category_Coverage ≥ 80%
```

**算法选择**：
- 预算 < 10W：贪心算法 + 动态规划
- 预算 10W-50W：遗传算法
- 预算 > 50W：整数规划 + 模拟退火

#### 3.2.4 风险过滤

**风险类型及处理**：

| 风险类型 | 检测方法 | 处理策略 |
|----------|----------|----------|
| 刷量嫌疑 | 互动异常检测、粉丝增长曲线分析 | 标记并降低排名 |
| 舆情风险 | 负面关键词监控、历史违规记录 | 排除或降权 |
| 档期冲突 | 竞品投放监测、档期查询 | 标记不可用时间 |
| 品类冲突 | 竞品合作历史检查 | 排除或提醒 |
| 未成年内容 | 内容审核、粉丝年龄分析 | 标记合规风险 |

## 四、Layer 3：内容创作 Agent（Content Creator）

### 4.1 Agent 定位

内容创作 Agent 是系统的**创意输出 Agent**，负责为每个推荐的 KOL 生成定制化的内容脚本。它确保内容既符合品牌调性，又能发挥 KOL 的个人风格。

### 4.2 核心功能模块

#### 4.2.1 脚本生成（Script Generation）

**输入信息**：
- 品牌信息（名称、产品卖点、品牌调性）
- KOL 信息（名称、风格、粉丝画像、历史爆款内容）
- 营销目标（曝光/种草/转化）
- 内容形式（短视频/图文/直播）
- 平台要求（TikTok/Instagram/小红书 等）

**脚本结构**：
```
短视频脚本 (60秒版本):
├── Hook (0-3秒): 吸引注意力
│   └─ "90%的人都选错了..."
├── Problem (3-10秒): 痛点引入
│   └─ "你是不是也有这种困扰..."
├── Solution (10-30秒): 产品展示
│   └─ "今天给大家介绍..."
├── Demo (30-50秒): 使用演示
│   └─ "看，我用了两周后..."
└── CTA (50-60秒): 行动号召
    └─ "点击下方链接..."
```

**LLM Prompt 设计**：
```
你是专业的短视频脚本编剧。请根据以下信息，为 KOL {kolName} 创作一条{campaignType}视频脚本。

## 品牌信息
- 品牌名称: {brandName}
- 产品卖点: {productHighlights}
- 品牌调性: {brandTone}
- 禁止内容: {avoidContent}

## KOL 信息
- KOL 名称: {kolName}
- KOL 风格: {kolStyle}
- 粉丝画像: {audienceProfile}
- 历史爆款: {topContents}

## 创作要求
- 时长: {duration}秒
- 目标: {marketingGoal}
- 平台: {platform}
- 语言: {language}

## 输出格式
请生成包含以下部分的脚本：
1. 核心信息点 (Key Messages)
2. 脚本正文 (Script)
3. 镜头建议 (Shot List)
4. 字幕/关键词 (Captions/Hashtags)
5. BGM 建议

确保脚本：
- 符合 KOL {kolName} 的个人风格
- 体现品牌调性 {brandTone}
- 包含明确的产品卖点
- 有强力的行动号召
```

#### 4.2.2 多语言适配（Multi-language Adaptation）

**支持语言**：
- 中文（简体/繁体）
- 英语（美式/英式）
- 日语、韩语、西班牙语、葡萄牙语 等

**适配要点**：
- 语义等效而非直译
- 文化敏感性调整
- 当地流行梗/热点融入
- 平台特性适配（TikTok 热梗、小红书 文风）

#### 4.2.3 合规检测（Compliance Detection）

**检测项目**：
- 广告标识合规（#ad、#sponsored）
- 功效声称合规（避免"第一"、"最强"等违规词）
- 平台规则合规（敏感词、限流风险）
- 版权内容检测（音乐、图片使用权限）

**检测流程**：
```
1. 敏感词过滤 → 广告法 + 平台规则词库
2. 声称合规检查 → 功效声称标准对照
3. 版权检测 → 背景音乐/素材版权查询
4. 风险评分 → 综合风险评估 (0-100)
```

**变体策略**：

## 五、Layer 4：效果优化 Agent（Performance Optimizer）

### 5.1 Agent 定位

效果优化 Agent 是系统的**决策支持 Agent**，负责整合前三个 Agent 的输出，进行效果预测和策略优化。它是整个系统的"战略大脑"，为投放决策提供数据支撑。

### 5.2 核心功能模块

#### 5.2.1 ROI 预测（ROI Prediction）

**预测模型**：
```
Expected_ROI = Σ (KOL_i.预测曝光 × KOL_i.预测转化率 × 平均客单价) / Σ KOL_i.成本

其中:
- 预测曝光 = KOL.历史平均曝光 × 品类系数 × 竞争系数
- 预测转化率 = KOL.历史CVR × 产品品类CVR × 脚本优化系数
```

**预测准确性**：
- 基于历史投放数据的回归模型
- 持续学习优化（数据飞轮）
- 置信区间展示（80%/90%/95%）

#### 5.2.2 预算分配优化（Budget Allocation）

**优化策略**：

| 策略名称 | 目标 | 适用场景 |
|----------|------|----------|
| 曝光最大化 | 最高总曝光 | 品牌曝光期 |
| ROI 最大化 | 最高投资回报 | 效果投放期 |
| 均匀分配 | 风险分散 | 测试阶段 |
| 头部集中 | 头部效应 | 冲刺GMV |

**预算分配示例**：

#### 5.2.3 风险评估（Risk Assessment）

**风险类型**：

| 风险类型 | 概率 | 影响 | 应对措施 |
|----------|------|------|----------|
| KOL 档期冲突 | 中 | 高 | 准备备选 KOL |
| 内容审核不过 | 低 | 高 | 多版本脚本准备 |
| 竞品同期投放 | 高 | 中 | 差异化内容策略 |
| 舆情危机 | 低 | 极高 | 实时监控 + 应急预案 |
| 平台政策变化 | 中 | 中 | 多平台分发 |
